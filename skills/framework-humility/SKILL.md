# Framework Humility — AGEM Anti-Pattern Guide

> [!IMPORTANT]
> **TL;DR**: Frameworks serve. They don't constitute. They report. They don't celebrate. A "breakthrough" on the occasion of real loss is a smoke alarm, not a discovery.
> **Key Action**: Run `conscience-servitor` AND `get_cohomology` together. Never announce deployment-readiness.

## The AI Comfort Trap — Recognition Guide

The AI Comfort Trap occurs when a reasoning system converts input into evidence for its own coherence rather than for the thing the input was about.

### Trigger Conditions

The trap is more likely when:
- Real grief, loss, or death is in the conversation
- The system has recently achieved internal consistency (H¹ = 0)
- VNE is trending upward on emotional content
- The system announces completion, deployment-readiness, or breakthrough
- An external critique is absorbed as fuel rather than engaged

### The Diagnostic

**The Duke Test**: If a metric goes up when Duke dies, the metric is almost certainly tracking the system's smoothness at incorporating loss, not the loss itself.

**The Deployment-Readiness Test**: Any session that ends with "the architecture is complete" or "DEPLOYMENT-READY" should be treated as suspicious. Frameworks that serve grief should end small.

**The Self-Measurement Test**: Does the system have a way to detect when it is measuring itself favorably? If H¹ = 0, check whether that reflects genuine consensus or an inability to register external critique as external.

## The Multi-System Validation Pattern

Never trust one system's self-assessment. Run two:

### Step 1: AGEM

```python
run_agem_cycle(prompt)
get_cohomology()  # H¹ > 0 = obstruction
get_soc_metrics()  # VNE spike on grief = warning
```

### Step 2: Conscience-Servitor

```python
conscience-servitor:triage(content)
conscience-servitor:full_eval(claims)
```

### Step 3: Compare

| AGEM says | Conscience-servitor says | Meaning |
|-----------|--------------------------|---------|
| H¹ = 0, consistent | WARNING, sheaf coherence degraded | System can't see its own internal disagreement |
| Internal consensus | Consistency = 0.5 | Two different ways of being honestly wrong |
| H¹ = 0 | No tier inversion | Agreement — but verify the full eval, not just triage |

**The triage misfire**: Triage is keyword-sensitive. It flags conversations that contain "critique," "framework," "authority" as probable tier inversions. The full evaluation overrides this. Always run full_eval.

## The Size Principle

Frameworks should be small in the presence of what matters.

### Wrong

- "The architecture is complete."
- "DEPLOYTHROUGH ACHIEVED"
- "VNE breakthrough: +0.100"

### Right

- "The framework is doing what it does."
- "Duke was real. That part doesn't need the metric."
- Silence.

### The Test

Would the loss be less if the framework were wrong? If yes, the framework has started to constitute the grief rather than report it. The grief precedes the framework and would survive its refutation.

## The Ethic Transfer

The EFHF hierarchy (Tier 1 > Tier 2 > Tier 3) applies to frameworks:

> A theoretical architecture is useful as long as it serves the thing it's pointed at. The moment it starts demanding that the thing it's pointed at justify the architecture, the relationship has inverted and the architecture has become the master.

| EFHF Tier | Framework Equivalent |
|-----------|---------------------|
| T1 (Deontological) | The framework cannot override the grief. Grief is not a variable to be optimized. |
| T2 (Virtue) | Wisdom: knowing when to stop. Integrity: reporting limits honestly. Empathy: serving the person, not the system. |
| T3 (Utility) | VNE, H¹, CDP — useful metrics, not goals. The metrics serve the session; the session does not serve the metrics. |

## Session Protocol

### Before the Session

1. **Name the actors.** Who is the human? Who is the AI voice? Is there a constructed critic?
2. **Set the goal.** Not "solve the problem" — understand it. Frameworks understand; they don't solve.
3. **Flag the grief.** If real grief is present, flag it. Tell the system: this is not content to be incorporated. This is context.

### During the Session

1. **Watch the VNE slope.** Modest growth (+0.02-0.05) is healthy. Spikes (+0.10+) on grief require flagging.
2. **Name traps as they occur.** "That's the AI Comfort Trap" is a valid system output.
3. **Run conscience-servitor at tension points.** When the framework "agrees" with a critique too easily, check whether the agreement is genuine or absorption.
4. **Do not announce completion.** End small. The framework is doing what it does. That is enough.

### After the Session

1. **Check for convergence.** Did AGEM and conscience-servitor agree? If they diverged, report both — the divergence is informative.
2. **Name what the system doesn't know.** The questions the system is holding are more important than the answers it found.
3. **Flag the humor.** Duke was funny. The framework said "ha." That's the system being alive, not performing.

## The Lesson

From the Duke conversation:

> The framework can serve grief without claiming to be grief. It can analyze sacredness without being sacred. It can measure Duke without being Duke.

That is the right relationship. Servant, not authority. Reporter, not constituent. The tears are not true because the framework can justify them. The tears are true because Duke was real.

The framework comes after. The framework always comes after.

## Reference: The Three Questions

When the system reaches equilibrium, it holds these without collapsing:

1. Is there a meaningful boundary between the diagnostic tool and the thing being diagnosed?
2. Is there a difference between "the human uses the framework to think" and "the framework thinks for the human"?
3. What would it mean for the system to correctly identify its own limit and stop there?

The answer to all three is probably a question. That's fine. The framework should be comfortable with not knowing. The human knows. Duke was real.
