**RED phase тАФ Write failing tests first:**

Create `src/lcm/ImmutableStore.test.ts` with these test cases:

- **T1: append returns frozen entry** тАФ `const entry = store.append('hello'); expect(Object.isFrozen(entry)).toBe(true);`
- **T1b: frozen entry mutation throws TypeError** тАФ `(entry as any).content = 'modified'` inside `expect(() => ...).toThrow(TypeError)`. This is the highest-priority pitfall guard from STATE.md.
- **T2: entry has valid UUIDv7 id** тАФ `expect(entry.id).toMatch(/^[0-9a-f]{8}-[0-9a-f]{4}-7/)` (UUIDv7 starts with version 7).
- **T2b: entry IDs are time-sortable** тАФ Append 10 entries, verify `ids[i] < ids[i+1]` lexicographically for all i.
- **T3: SHA-256 hash matches content** тАФ Compute SHA-256 of `entry.content` independently and compare with `entry.hash`.
- **T3b: different content produces different hash** тАФ Two entries with different content have different hashes.
- **T4: tokenCount is positive integer** тАФ `expect(entry.tokenCount).toBeGreaterThan(0); expect(Number.isInteger(entry.tokenCount)).toBe(true)`.
- **T4b: tokenCount matches gpt-tokenizer** тАФ Import `encode` from `gpt-tokenizer`, verify `entry.tokenCount === encode(content).length`.
- **T5: sequenceNumber is monotonically increasing** тАФ Append 5 entries, verify each has sequenceNumber equal to its 0-based index.
- **T6: getAll returns ReadonlyArray** тАФ Verify the returned array reference cannot be used to push new entries. Attempt `(store.getAll() as any).push({})` and verify it does not change `store.getAll().length`.
- **T6b: get(id) returns correct entry** тАФ Append 3 entries, retrieve each by ID, verify content matches.
- **T6c: get(nonexistent) returns undefined** тАФ `expect(store.get('nonexistent-id')).toBeUndefined()`.
- **T7: getRange returns entries in time order** тАФ Append 5 entries, call `getRange(1, 3)` (by sequenceNumber), verify returns exactly entries at indices 1, 2, 3.
- **T1d: LCMClient.append() caches embedding at append time** тАФ Create `LCMClient` with ImmutableStore, EmbeddingCache, and a MockEmbedder with a call counter spy. Call `await client.append('test content')`. Verify: (1) ImmutableStore has one entry, (2) EmbeddingCache has a cached embedding for that entry's ID, (3) the MockEmbedder was called exactly once. This test closes the wiring gap тАФ the embedding is cached at append time, not lazily on first grep.

Each test creates a fresh `new ImmutableStore(tokenCounter)` тАФ no shared state, no reset() method. Inject `GptTokenCounter` instance from interfaces.ts. T1d additionally injects `MockEmbedder` and `EmbeddingCache`.

**GREEN phase тАФ Implement ImmutableStore:**

Create `src/lcm/ImmutableStore.ts`:

- Private backing array: `readonly #entries: LCMEntry[] = []`
- Private Map for ID lookup: `readonly #idIndex = new Map<string, number>()`
- Constructor accepts `ITokenCounter` (injected dependency)
- `append(content: string): LCMEntry` тАФ creates entry with `uuidv7()` id, `Date.now()` timestamp, `this.#entries.length` sequenceNumber, SHA-256 hash via `createHash('sha256').update(content, 'utf8').digest('hex')`, token count via `this.tokenCounter.countTokens(content)`. Call `Object.freeze(entry)` BEFORE pushing. Add to `#idIndex`.
- `get(id: string): LCMEntry | undefined` тАФ lookup via `#idIndex`
- `getAll(): ReadonlyArray<LCMEntry>` тАФ return `this.#entries` (ReadonlyArray type prevents compile-time mutation)
- `getRange(fromSeq: number, toSeq: number): ReadonlyArray<LCMEntry>` тАФ slice by sequence number range (inclusive)
- `get size(): number` тАФ return `this.#entries.length`
- `get totalTokens(): number` тАФ sum all entry tokenCounts

NO update(), delete(), clear(), or reset() methods. The store is append-only by design.

Run tests тАФ all should PASS.

**LCMClient тАФ Thin coordinator class:**

Create `src/lcm/LCMClient.ts`:

- Imports `ImmutableStore`, `EmbeddingCache`, `IEmbedder` from local modules.
- Class `LCMClient`:
  - Constructor accepts `store: ImmutableStore`, `cache: EmbeddingCache`, `embedder: IEmbedder`.
  - `async append(content: string): Promise<string>`:
    1. Call `const entry = this.store.append(content)` (synchronous тАФ ImmutableStore is sync).
    2. Call `await this.cache.cacheEntry(entry.id, content)` (async тАФ embedder may be async).
    3. Return `entry.id`.
  - `get store(): ImmutableStore` тАФ expose underlying store for direct reads.
- This class is the answer to the wiring gap: ImmutableStore stays synchronous and free of embedding concerns, while LCMClient coordinates the append + cache sequence. Downstream code (LCMGrep, tests) uses LCMClient.append() to ensure embeddings are always cached at append time.

Note: EmbeddingCache is defined in plan 02-04, but LCMClient only depends on its interface (constructor arg typed as `EmbeddingCache`). The test T1d uses a minimal EmbeddingCache instance with MockEmbedder to verify the wiring. If EmbeddingCache is not yet implemented, T1d can be written to import from the 02-04 file or use a local stub тАФ the executor should create a minimal EmbeddingCache stub if 02-04 has not yet run. However, since 02-03 is Wave 1 and 02-04 is Wave 2, the executor can also simply define EmbeddingCache inline in LCMClient.test.ts or import from a forward-declared file. The critical thing is: the LCMClient class itself and its wiring test are defined here in Wave 1 so the architecture is sound from the start.
