**GREEN phase — Implement EmbeddingCache and LCMGrep:**

Create `src/lcm/EmbeddingCache.ts`:

- Constructor accepts `IEmbedder`.
- Private `#cache: Map<string, Float64Array>` — maps entry ID to embedding vector.
- `async cacheEntry(entryId: string, content: string): Promise<void>` — calls `embedder.embed(content)`, stores result. Called by LCMClient.append() at append time (see 02-03 LCMClient). LCMGrep falls back to caching on first grep if not yet cached (hybrid strategy).
- `getEmbedding(entryId: string): Float64Array | undefined` — returns cached embedding.
- `async refreshEntry(entryId: string, content: string): Promise<void>` — forces recomputation.
- `has(entryId: string): boolean`

Create `src/lcm/LCMGrep.ts`:

- Export interface `GrepResult`: `{ entry: LCMEntry; similarity: number }`.
- Export interface `GrepOptions`: `{ maxResults?: number; minSimilarity?: number }`.
- Class `LCMGrep`:
  - Constructor accepts `ImmutableStore`, `EmbeddingCache`, `IEmbedder`.
  - `async grep(query: string, options?: GrepOptions): Promise<GrepResult[]>`:
    1. Embed the query via `this.embedder.embed(query)`.
    2. For each entry in the store, get its cached embedding from EmbeddingCache. If not cached, compute and cache it now (hybrid strategy per user decision).
    3. Compute cosine similarity: `dot(queryVec, entryVec) / (norm(queryVec) * norm(entryVec))`. Since MockEmbedder and real embedder both normalize, this simplifies to `dot(queryVec, entryVec)`.
    4. Filter by `minSimilarity` (default: -1, meaning all entries).
    5. Sort by similarity descending.
    6. Return top `maxResults` (default: all).
  - Helper `cosineSimilarity(a: Float64Array, b: Float64Array): number` — pure function, no external deps.
  - `async cacheAllEntries(): Promise<void>` — pre-caches all store entries. Called during initialization or manually.

**REFACTOR:** Ensure cosine similarity is a standalone pure function. Verify no direct import of @huggingface/transformers anywhere in LCMGrep.ts or EmbeddingCache.ts.
