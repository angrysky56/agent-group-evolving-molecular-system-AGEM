**GREEN phase — Implement EscalationProtocol:**

Create `src/lcm/EscalationProtocol.ts`:

- Import `EventEmitter` from `node:events`.
- Import `encode, decode` from `gpt-tokenizer` (for L3 deterministic truncation).
- Import `ICompressor, ITokenCounter, EscalationThresholds, EscalationLevel` from `interfaces.js`.

Class `EscalationProtocol extends EventEmitter`:

Constructor:

- `compressor: ICompressor` (injected — for L1 and L2)
- `tokenCounter: ITokenCounter` (injected — for counting)
- `thresholds: EscalationThresholds` (initial values)

Methods:

- `setThresholds(partial: Partial<EscalationThresholds>): void` — merges with existing thresholds. Runtime-adjustable per user decision.
- `async escalate(text: string): Promise<EscalationResult>`:
  1. Count tokens. If below `level1TokenLimit`, return as-is (no escalation needed).
  2. **Level 1 (nuanced compression):** Call `compressor.compress(text, targetRatio)` where targetRatio = `level1TokenLimit / inputTokens`. Check output token count. If `outputTokens / inputTokens <= level2MinRatio`, return L1 result. Emit `'escalation'` event with level 1.
  3. **Level 2 (multi-compression indexing):** Chunk the input into segments (by paragraph or fixed token chunks). Compress each segment independently via compressor. Concatenate compressed segments. Check total token count. If sufficient compression achieved, return L2 result. Emit event.
  4. **Level 3 (deterministic chunking + truncation fallback):**
     a. **Chunk step:** Split the input text into logical sections by splitting on double newlines (`\n\n`). For each chunk, apply deterministic compression: take the first N% of each chunk's tokens (same approach as MockCompressor — no LLM call). This preserves the semantic structure of the input by keeping compressed representations of each section.
     b. **Concatenate:** Join the compressed chunks with double newline separators.
     c. **Check:** If the concatenated result's token count is <= `level3KTokens`, return it as the L3 result.
     d. **Hard truncation fallback:** If still over budget, apply `deterministicTruncate(concatenated, level3KTokens)`.
     e. Return L3 result with `chunks: numberOfChunks` in metadata. Emit event. **No compressor.compress() call anywhere in L3 — all compression is deterministic (token slicing).**

Type `EscalationResult`: `{ level: EscalationLevel; output: string; inputTokens: number; outputTokens: number; compressorUsed: boolean; chunks?: number }`.

Private `deterministicChunkCompress(text: string, kTokens: number): { output: string; chunks: number }`:

- Split `text` on `\n\n` into chunks. If only one chunk (no double newlines), treat the whole text as a single chunk.
- For each chunk: `const tokens = encode(chunk); const compressed = decode(tokens.slice(0, Math.ceil(tokens.length * 0.5)));` (50% compression ratio — deterministic, no LLM).
- Concatenate compressed chunks with `\n\n`.
- If total tokens <= kTokens, return `{ output: concatenated, chunks: chunks.length }`.
- Otherwise, apply hard truncation: `return { output: decode(encode(concatenated).slice(0, kTokens)), chunks: chunks.length }`.
- This function MUST NOT call `this.compressor` — it is the hard exit path. All compression is done via token slicing.

Private `deterministicTruncate(text: string, kTokens: number): string`:

- `const tokens = encode(text);`
- `if (tokens.length <= kTokens) return text;`
- `return decode(tokens.slice(0, kTokens));`
- This is the final fallback within `deterministicChunkCompress`. Also available standalone.

**REFACTOR:** Ensure `deterministicChunkCompress` and `deterministicTruncate` are clearly isolated and cannot accidentally invoke the compressor. Consider making them standalone exported functions for testability.

**GREEN phase — Implement lcm_expand:**

Create `src/lcm/LCMExpand.ts`:

Export `async function* lcm_expand(summaryNodeId: string, dag: ContextDAG): AsyncGenerator<ExpandLevel, void, unknown>`:

1. Get the SummaryNode from the DAG: `dag.getSummaryNode(summaryNodeId)`.
2. If not found, return (generator ends immediately).
3. **Level 0 — Summary:** `yield { kind: 'summary', nodeId: summaryNodeId, content: summary.content }`.
4. **Level 1+ — Intermediate compressions (if any):** For each `compression` in `summary.intermediateCompressions`, yield `{ kind: 'compression', level: compression.level, content: compression.content, pointsTo: compression.childIds }`.
5. **Level N — Original entries:** For each `entryId` in `summary.originalEntryIds`, resolve via `dag.getEntry(entryId)`. If found, yield `{ kind: 'entry', entryId, content: entry.content, tokenCount: entry.tokenCount }`.

The key property: entries are yielded one-at-a-time via the async generator. If the consumer breaks early, remaining entries are never fetched. This is the lazy/streaming pattern per user decision.

Import `ExpandLevel` from `interfaces.js`, `ContextDAG` from `ContextDAG.js`.

**REFACTOR:** Ensure the function is a clean async generator with no eagerly-loaded arrays. The entire hierarchy traversal should be lazy.
