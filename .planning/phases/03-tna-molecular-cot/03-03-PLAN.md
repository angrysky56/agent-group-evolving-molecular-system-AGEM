**GREEN phase — Implement GapDetector:**

Create `src/tna/GapDetector.ts`:

- Class `GapDetector`:
  - Constructor accepts `CooccurrenceGraph`, `LouvainDetector`, `CentralityAnalyzer`.
  - Private `gaps: GapMetrics[] = []`.
  - `findGaps(): ReadonlyArray<GapMetrics>`:
    1. Get community assignments from `louvainDetector.getAssignment()` for all nodes.
    2. Get the list of distinct community IDs.
    3. If there is only 1 community, return [] (no gaps possible — ROADMAP criterion 3 first edge case).
    4. For each pair of communities (i, j) where i < j:
       a. Count inter-community edges: edges where one endpoint is in community i and the other in community j.
       b. Compute inter-community density: `interEdges / (|community_i| * |community_j|)`.
       c. If density is below a threshold (e.g., `density < 0.2` — configurable), this is a structural gap.
       d. Compute shortest path length: average shortest path between a sample of nodes from community i to community j. Use BFS on the graphology graph (graphology provides traversal utilities, or implement BFS directly — it's simple on an undirected graph).
       e. Compute modularity delta: estimate how much modularity Q would change if communities i and j were merged. `modularityDelta = Q_separate - Q_merged` (approximate via: if there are few inter-community edges relative to intra, merging would decrease Q, so delta is positive).
       f. Identify bridge nodes: nodes in either community that have betweenness centrality above the median centrality AND have at least one inter-community edge. Use the CentralityAnalyzer scores.
    5. Store gaps sorted by inter-community density (ascending — lowest density = biggest gap).
    6. Return gaps.
  - `findNearestGap(): GapMetrics | undefined` — returns the gap with the lowest inter-community density (the most promising exploration target). Returns undefined if no gaps.
  - `getGapCount(): number` — number of detected gaps.
  - `getGapBetween(communityA: number, communityB: number): GapMetrics | undefined` — specific gap lookup.

**Design note on gap detection algorithm:**
The key insight is that structural gaps are LOW-density inter-community regions, not zero-density. If two communities have zero edges between them, they are simply disconnected — not a "gap" in the semantic sense. A gap exists when there is weak (but non-zero) connection that could be strengthened by exploration. However, for the canonical test case (two-clique bridge), even a single bridge edge creates a gap because the density is extremely low relative to intra-community density.

For T18b (three-cluster graph with A-C disconnected): Whether A-C counts as a gap depends on the definition. If we define gap as "community pair with below-threshold density AND at least one inter-community edge", then A-C (zero edges) is NOT a gap — it's a disconnection. Only A-B and B-C (with their single bridge edges) are gaps. This is the more useful definition for Van der Waals targeting because exploration needs at least some existing connection to bridge from.
