**Create `src/tna/interfaces.ts`** with all TNA domain types:

Types to define:

- `TextNodeId` — branded string type: `string & { readonly __brand: 'TextNodeId' }` (same pattern as VertexId/EdgeId in GraphTypes.ts). The ID IS the canonical lemma form of the concept.
- `TextNode` interface: `readonly id: TextNodeId`, `readonly lemma: string` (canonical form), `readonly surfaceForms: readonly string[]` (all observed morphological variants), `readonly tfidfWeight: number`, `readonly communityId?: number` (set by LouvainDetector), `readonly betweennessCentrality?: number` (set by CentralityAnalyzer).
- `TextEdge` interface: `readonly source: TextNodeId`, `readonly target: TextNodeId`, `readonly weight: number` (co-occurrence weight from 4-gram window), `readonly createdAtIteration: number` (required for Phase 4 SOC surprising-edge-ratio per-iteration computation — Pitfall 9 prevention).
- `CommunityAssignment` interface: `readonly nodeId: TextNodeId`, `readonly communityId: number`, `readonly modularity: number`.
- `GapMetrics` interface: `readonly communityA: number`, `readonly communityB: number`, `readonly interCommunityDensity: number`, `readonly shortestPathLength: number`, `readonly modularityDelta: number`, `readonly bridgeNodes: readonly TextNodeId[]`.
- `TNAConfig` interface with defaults: `windowSize: number` (default 4), `minTfidfWeight: number` (default 0.1), `louvainSeed: number` (default 42), `language: string` (default 'en').
- `PreprocessResult` interface: `readonly tokens: readonly string[]` (lemmatized, stopword-removed tokens), `readonly tfidfScores: ReadonlyMap<string, number>`.

Use `.js` extensions in all imports (NodeNext module resolution, matching Phase 1/2 convention).

**Create `src/types/MolecularCoT.ts`** with bond type interfaces that enforce behavioral invariants at the type-system level per locked decision:

- `StepId` — branded string type: `string & { readonly __brand: 'StepId' }`.
- `BondType` — string literal union: `'covalent' | 'hydrogen' | 'vanDerWaals'`.

Bond interfaces with BEHAVIORAL INVARIANTS (not just metadata):

- `CovalentBond` interface:
  - `readonly type: 'covalent'`
  - `readonly source: StepId`
  - `readonly target: StepId`
  - `readonly strength: number` (0.0 to 1.0)
  - Behavioral: removing a covalent bond MUST trigger `cascadeInvalidate(target)` that marks all transitively dependent steps as requiring re-derivation.

- `HydrogenBond` interface:
  - `readonly type: 'hydrogen'`
  - `readonly source: StepId`
  - `readonly target: StepId`
  - `readonly semanticDistance: number`
  - Behavioral: creation MUST be rejected if `semanticDistance > HYDROGEN_DISTANCE_THRESHOLD`.

- `VanDerWaalsBond` interface:
  - `readonly type: 'vanDerWaals'`
  - `readonly source: StepId`
  - `readonly target: StepId`
  - `readonly trajectoryLength: number`
  - Behavioral: creation MUST be rejected if `trajectoryLength < VAN_DER_WAALS_MIN_TRAJECTORY` (5.0 per paper's 5.32 average).

- `MolecularBond` — discriminated union: `CovalentBond | HydrogenBond | VanDerWaalsBond`.

- Constants: `HYDROGEN_DISTANCE_THRESHOLD = 0.7` (configurable), `VAN_DER_WAALS_MIN_TRAJECTORY = 5.0`.

- `BondGraph` class that ENFORCES behavioral invariants:
  - `addCovalentBond(source, target, strength): CovalentBond` — always succeeds.
  - `addHydrogenBond(source, target, semanticDistance): HydrogenBond` — throws if `semanticDistance > HYDROGEN_DISTANCE_THRESHOLD`.
  - `addVanDerWaalsBond(source, target, trajectoryLength): VanDerWaalsBond` — throws if `trajectoryLength < VAN_DER_WAALS_MIN_TRAJECTORY`.
  - `removeCovalentBond(source, target): StepId[]` — removes the bond AND returns all transitively dependent step IDs that need re-derivation (cascadeInvalidate). Uses DFS from target through all covalent bonds to find transitive dependents.
  - `getBonds(): ReadonlyArray<MolecularBond>` — all bonds.
  - `getDependents(stepId: StepId): ReadonlyArray<StepId>` — all transitive dependents via covalent bonds.
  - Internal storage: `Map<string, MolecularBond>` keyed on `${source}:${target}`.

This is NOT metadata tagging — the BondGraph class makes it impossible to create bonds that violate invariants. The type system + runtime checks together enforce the behavioral constraints.

**GREEN phase — Implement Preprocessor:**

Create `src/tna/Preprocessor.ts`:

- Class `Preprocessor`:
  - Constructor accepts optional `TNAConfig` (defaults applied).
  - Private `tfidf` instance from `natural.TfIdf`.
  - Private lemmatization function using `wink-lemmatizer` (verb(), noun(), adjective() methods — try verb first, then noun, then adjective; take the shortest result as canonical form). If wink-lemmatizer does not cover the word, fall back to Porter stemmer from `natural`.
  - `lemmatize(word: string): string` — returns canonical lemma form. Public for direct testing.
  - `preprocess(text: string): PreprocessResult`:
    1. Tokenize using `natural.WordTokenizer`.
    2. Lowercase all tokens.
    3. Remove stopwords using `stopword.removeStopwords()`.
    4. Lemmatize each remaining token.
    5. Compute TF-IDF scores (add document to TfIdf, extract scores).
    6. Filter tokens below `config.minTfidfWeight` threshold.
    7. Return `{ tokens, tfidfScores }`.
  - `addDocument(text: string): void` — adds a document to the TF-IDF corpus for multi-document scoring.
  - `preprocessWithCorpus(text: string): PreprocessResult` — preprocesses using accumulated corpus TF-IDF.

The CRITICAL invariant: lemmatization happens BEFORE any tokens are returned. No raw surface form ever reaches the graph.

Run Preprocessor tests — all should PASS.

**GREEN phase — Implement CooccurrenceGraph:**

Create `src/tna/CooccurrenceGraph.ts`:

- Class `CooccurrenceGraph`:
  - Constructor accepts `Preprocessor` instance and optional `TNAConfig`.
  - Private `graph` field: `new Graph({ type: 'undirected', multi: false })` from graphology.
  - Private `nodeMetadata: Map<string, TextNode>` — tracks full TextNode data per lemma.
  - Private `currentIteration: number = 0`.
  - `ingest(text: string, iteration?: number): void`:
    1. If iteration provided, set `currentIteration = iteration`.
    2. Call `this.preprocessor.preprocess(text)` to get lemmatized tokens.
    3. Apply 4-gram sliding window over tokens:
       - For each token at position i, look at tokens at positions i+1 through i+windowSize-1.
       - Edge weight = windowSize - distance (so adjacency-1 = 3, adjacency-2 = 2, adjacency-3 = 1 for window=4).
       - If edge already exists, ADD weight to existing edge weight.
    4. For each new lemma token: create graph node, create TextNode entry in nodeMetadata with the lemma as ID (TextNodeId). Track surfaceForms — the original pre-lemmatization token that produced this lemma.
    5. For each edge: set `createdAtIteration` attribute on graph edge.
  - `getGraph(): Graph` — return the underlying graphology instance (read access for LouvainDetector, CentralityAnalyzer).
  - `getNode(lemma: string): TextNode | undefined` — lookup node metadata.
  - `getNodes(): ReadonlyArray<TextNode>` — all nodes.
  - `getEdgeWeight(source: string, target: string): number` — edge weight or 0.
  - `get order(): number` — node count.
  - `get size(): number` — edge count.

The CRITICAL design: `ingest()` calls `Preprocessor.preprocess()` internally — raw text NEVER touches the graph without lemmatization. This is the pitfall guard.

Run CooccurrenceGraph tests — all should PASS.
