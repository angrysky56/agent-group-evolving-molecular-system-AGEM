**GREEN phase — Implement LouvainDetector:**

Create `src/tna/LouvainDetector.ts`:
- Class `LouvainDetector`:
  - Constructor accepts `CooccurrenceGraph` instance.
  - Private `assignments: Map<string, number>` — node -> community ID.
  - Private `modularity: number = 0`.
  - Private `communityCount: number = 0`.
  - `detect(seed?: number): { assignments: ReadonlyMap<string, number>; modularity: number; communityCount: number }`:
    1. Get the underlying graphology graph via `cooccurrenceGraph.getGraph()`.
    2. Call `louvain(graph, { resolution: 1, randomWalk: true, ...seedOptions })` from `graphology-communities-louvain`.
    3. **Deterministic seeding strategy:** The `graphology-communities-louvain` library's `louvain()` function accepts a `randomWalk` boolean. For determinism, we need to seed the random number generator. If the library supports a `rng` option (check the API), pass a seeded PRNG. If not, implement a seedable wrapper: use the seed to create a deterministic node visit order by sorting nodes and using a seeded shuffle (Mulberry32 or similar simple PRNG). The critical requirement is: same graph + same seed = same output. Test T9 validates this.
    4. Store assignments in `this.assignments`.
    5. Compute community count from unique assignment values.
    6. Return the result object.
  - `getAssignment(nodeId: string): number | undefined` — lookup.
  - `getCommunityMembers(communityId: number): ReadonlyArray<string>` — all nodes in that community.
  - `getCommunityCount(): number` — number of distinct communities.
  - `getModularity(): number` — last computed modularity.

**Important implementation note on determinism:** The `graphology-communities-louvain` library uses `Math.random()` internally for node visit randomization. To achieve deterministic seeding:
- Option A: If the library accepts an `rng` function parameter, pass a seeded Mulberry32 PRNG.
- Option B: If not, use the `louvain.detailed()` function which may expose more control. Check the actual API surface.
- Option C: If neither option works, implement a wrapper that replaces `Math.random` temporarily during the Louvain call with a seeded PRNG (save/restore). This is a common pattern for determinism in graph algorithms.
- Option D: Sort nodes deterministically before passing to Louvain, which combined with a fixed resolution parameter may produce deterministic results for structured graphs.

The T9 test is the gate — whichever approach produces 10/10 identical runs passes.

Run LouvainDetector tests — all should PASS.
