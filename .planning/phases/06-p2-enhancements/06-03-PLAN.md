```typescript
/**
 * CatalystQuestion — a targeted question designed to bridge two communities.
 *
 * Generated by CatalystQuestionGenerator for each structural gap.
 * Used as reasoning priors for VdW agents (ORCH-06).
 */
export interface CatalystQuestion {
  /** The structural gap this question targets. */
  readonly gapId: string;

  /** Community IDs being bridged. */
  readonly communityA: number;
  readonly communityB: number;

  /** The generated question text. */
  readonly questionText: string;

  /** Representative nodes from each community used as seed concepts. */
  readonly seedNodeA: string;
  readonly seedNodeB: string;

  /** Semantic distance between communities (0-1; 1 = maximum void). */
  readonly semanticDistance: number;

  /** Priority ranking (lower = higher priority). */
  readonly priority: number;
}

// ---------------------------------------------------------------------------
// Phase 6: Centrality time-series types (TNA-09)
// ---------------------------------------------------------------------------

/**
 * CentralityTrend — direction of centrality change over time.
 */
export type CentralityTrend = "rising" | "falling" | "stable" | "oscillating";

/**
 * CentralityTimeSeries — time-series centrality data for a single node.
 */
export interface CentralityTimeSeries {
  readonly nodeId: string;
  readonly scores: ReadonlyArray<{ iteration: number; score: number }>;
  readonly trend: CentralityTrend;
  readonly peak: { iteration: number; score: number } | null;
  readonly valley: { iteration: number; score: number } | null;
}

/**
 * CentralityTimeSeriesConfig — configuration for time-series tracking.
 */
export interface CentralityTimeSeriesConfig {
  /**
   * Default interval (iterations) between centrality recomputations.
   * Default: 10
   */
  readonly defaultComputeInterval: number;

  /**
   * Interval during transitioning/critical regimes.
   * Default: 5
   */
  readonly urgentComputeInterval: number;

  /**
   * Interval during stable regimes.
   * Default: 20
   */
  readonly relaxedComputeInterval: number;

  /**
   * Minimum trend slope magnitude to classify as rising/falling.
   * Default: 0.05
   */
  readonly trendSlopeThreshold: number;

  /**
   * Multiplier for rapid centrality change detection.
   * If score increases by this factor in 2 data points, emit event.
   * Default: 3.0 (3x increase)
   */
  readonly rapidChangeMultiplier: number;
}
```

**CatalystQuestionGenerator class:**

```typescript

import type { CentralityAnalyzer } from "./CentralityAnalyzer.js";
```

Purpose: For each structural gap between communities, extract representative nodes, compute semantic distance (approximated via graph topology in Phase 6; real embeddings in Phase 7), and generate template-based catalyst questions.

**Private fields:**


**Constructor:**

```typescript
constructor(
  cooccurrenceGraph: CooccurrenceGraph,
  centralityAnalyzer: CentralityAnalyzer,
  config?: { maxRepresentativeNodes?: number; maxQuestionsPerGap?: number }
) {
  this.#cooccurrenceGraph = cooccurrenceGraph;
  this.#centralityAnalyzer = centralityAnalyzer;
  this.#maxRepresentativeNodes = config?.maxRepresentativeNodes ?? 3;
  this.#maxQuestionsPerGap = config?.maxQuestionsPerGap ?? 3;
}
```

**Key methods:**

1. `generateQuestions(gap: GapMetrics): CatalystQuestion[]`
   - Build gapId: `${gap.communityA}_${gap.communityB}`
   - Check cache: if gapId exists in cache, return cached questions.
   - Extract representative nodes from community A and B.
   - Compute semantic distance (approximation).
   - Generate questions from templates.
   - Cache and return.

2. `#getRepresentativeNodes(communityId: number, count: number): TextNode[]`
   - Get all nodes from CooccurrenceGraph with matching communityId.
   - Sort by betweennessCentrality descending.
   - Return top N nodes.
   - If community has fewer than N nodes, return all available.

3. `#computeSemanticDistance(nodesA: TextNode[], nodesB: TextNode[]): number`
   - Phase 6 approximation: use graph-based distance as proxy.
   - Compute average TF-IDF weight difference between communities.
   - Normalize to [0, 1] range.
   - In Phase 7: would use actual embedding cosine similarity.
   - Fallback: if no nodes, return 1.0 (maximum distance = maximum void).

4. `#generateFromTemplates(nodesA: TextNode[], nodesB: TextNode[], gap: GapMetrics, distance: number): CatalystQuestion[]`
   - Template patterns:
     a. `"How does [nodeA.lemma] relate to [nodeB.lemma]?"` — direct relationship probe.
     b. `"What concept bridges [nodeA.lemma] and [nodeB.lemma]?"` — bridging concept probe.
     c. `"In what context would [nodeA.lemma] and [nodeB.lemma] co-occur?"` — co-occurrence probe.
   - For each pair of (nodeA[i], nodeB[i]) up to maxQuestionsPerGap:
     - Select template based on index (rotate through 3 templates).
     - Build CatalystQuestion with gapId, seeds, distance, priority.
   - Priority ordering: lower semantic distance → higher priority (more achievable bridges).

5. `generateBatchQuestions(gaps: ReadonlyArray<GapMetrics>): Map<string, CatalystQuestion[]>`
   - Process all gaps; return map of gapId → questions.
   - Useful for batch processing during obstruction handling.

6. `getCachedQuestions(gapId: string): CatalystQuestion[] | undefined`
   - Look up cache directly.

7. `invalidateCache(gapId?: string): void`
   - If gapId provided, remove specific entry.
   - If no gapId, clear entire cache.
   - Called when communities merge or gaps disappear.

8. `getCacheSize(): number`
   - Returns number of cached gap entries.

**Edge cases handled:**

- Community has < 3 nodes: use all available as representatives.
- Semantic distance is 0 (identical communities): generate reconciliation-style questions.
- No nodes in community: return empty questions array.
- Gap disappears: cache invalidation removes stale entries.

**Design constraints:**

- ZERO imports from lcm/, sheaf/, soc/, or orchestrator/.
- Imports only from sibling TNA files (CooccurrenceGraph, CentralityAnalyzer, interfaces).
- No LLM calls in Phase 6 (template-based only; LLM integration deferred to Phase 7).
- Cache uses simple Map<string, CatalystQuestion[]> (no TTL; manual invalidation).

**Default config:**

```typescript
const DEFAULT_TIMESERIES_CONFIG: CentralityTimeSeriesConfig = {
  defaultComputeInterval: 10,
  urgentComputeInterval: 5,
  relaxedComputeInterval: 20,
  trendSlopeThreshold: 0.05,
  rapidChangeMultiplier: 3.0,
};
```

**New public methods:**

1. `computeIfDue(iteration: number): boolean`
   - Check if currentIteration - lastComputeIteration >= currentInterval.
   - If due: call compute() (existing method), then updateTimeSeries(iteration).
   - Return true if computation was performed, false if skipped.
   - This is the main entry point called each iteration by the orchestrator.

2. `updateTimeSeries(iteration: number): void`
   - For each node with a centrality score:
     - Append { iteration, score } to the time-series map.
     - Trim to keep at most 50 data points per node (prevent unbounded growth).
   - Detect rapid changes:
     - For each node, compare latest score to previous score.
     - If latest >= previous \* rapidChangeMultiplier: emit 'tna:centrality-change-detected'.
   - Detect major topology reorganization:
     - Count nodes whose rank changed by > 50% of total nodes.
     - If count > 3: emit 'tna:topology-reorganized'.
   - Update lastComputeIteration.

3. `adjustInterval(regime: string): void`
   - If regime === 'transitioning' or regime === 'critical': set interval to urgentComputeInterval.
   - If regime === 'stable': set interval to relaxedComputeInterval.
   - Otherwise: set interval to defaultComputeInterval.

4. `getTimeSeries(nodeId: string): CentralityTimeSeries | undefined`
   - Build and return CentralityTimeSeries for the given node.
   - Compute trend from scores array.
   - Find peak and valley.
   - Return undefined if no data for this node.

5. `getAllTimeSeries(): ReadonlyArray<CentralityTimeSeries>`
   - Return time-series for all tracked nodes.

6. `getRisingNodes(minSlope?: number): ReadonlyArray<CentralityTimeSeries>`
   - Return nodes with 'rising' trend.
   - Optional minSlope filter.

7. `#computeTrend(scores: Array<{ iteration: number; score: number }>): CentralityTrend`
   - If fewer than 3 data points: return 'stable'.
   - Take the last 3 data points.
   - Compute slope: (last.score - first.score) / 2.
   - If |slope| < trendSlopeThreshold: return 'stable'.
   - If slope > trendSlopeThreshold: return 'rising'.
   - If slope < -trendSlopeThreshold: return 'falling'.
   - Check for oscillation: if the sign of consecutive deltas alternates, return 'oscillating'.

8. `#findPeak(scores: Array<{ iteration: number; score: number }>): { iteration: number; score: number } | null`
   - Return entry with highest score. Null if empty.

9. `#findValley(scores: Array<{ iteration: number; score: number }>): { iteration: number; score: number } | null`
   - Return entry with lowest score. Null if empty.

**Backward compatibility:**

- The existing compute(), getScore(), getTopNodes(), getBridgeNodes() methods are UNCHANGED.
- The EventEmitter extension is additive (CentralityAnalyzer already has no base class).
- Existing tests call compute() directly and do not check for EventEmitter — they will still pass.
- The new time-series methods are all additive.

**Design constraints:**

- NO imports from lcm/, sheaf/, soc/, or orchestrator/.
- Event emission uses EventEmitter (same pattern as CohomologyAnalyzer, SOCTracker).
- ComposeRootModule will wire these events to EventBus (same pattern as sheaf/soc events).
