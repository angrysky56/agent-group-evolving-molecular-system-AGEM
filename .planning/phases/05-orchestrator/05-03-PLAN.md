ObstructionHandler class:

- Purpose: Listen to Sheaf obstruction events (H^1 detection) and spawn gapDetector agents to fill semantic voids

1. Private fields:
   - #eventBus: EventBus
   - #gapDetector: GapDetector (from TNA)
   - #tnaGraph: CooccurrenceGraph (context for agents)
   - #agentPool: AgentPool
   - #obstructionQueue: ObstructionEvent[] = [] (pending obstructions to process)
   - #isProcessing: boolean = false (lock to serialize processing)

2. Constructor(eventBus: EventBus, gapDetector: GapDetector, tnaGraph: CooccurrenceGraph, config?: { agentPoolSize?: number })
   - Store eventBus, gapDetector, tnaGraph
   - Create AgentPool with GapDetectorAgent factory (stub agents for Phase 5)
   - Subscribe to 'sheaf:h1-obstruction-detected' event
   - On obstruction event: enqueue and process asynchronously

3. Private onObstructionDetected(event: SheafEvent): void
   - Called when 'sheaf:h1-obstruction-detected' fires
   - Append event to #obstructionQueue
   - Call this.#processQueue()

4. Private async #processQueue(): Promise<void>
   - If #isProcessing already true, return (already processing)
   - While #obstructionQueue length > 0:
     a. Set #isProcessing = true
     b. Dequeue next obstruction event
     c. Get idle agent from agentPool: const agent = this.#agentPool.getIdleAgents()[0]
     d. If no idle agents available, create new agent in pool (up to config.agentPoolSize)
     e. Spawn agent for obstruction filling:
     ```typescript
     const gapFillResult = await this.#runGapDetectorAgent(
       agent,
       obstructionEvent,
     );
     ```
     f. Feed results back to TNA graph:
     ```typescript
     this.#integrateGapFillResults(gapFillResult);
     ```
     g. Mark agent idle
     h. Emit 'orch:obstruction-filled' event with results
     i. Set #isProcessing = false

5. Private async #runGapDetectorAgent(agent: Agent, obstruction: SheafEvent): Promise<GapFillResult>
   - Extract obstruction context: H^1 dimension, communities involved
   - Call gapDetector.detectGaps() to identify semantic voids
   - For each gap:
     a. Generate synthetic prompt/query to fill the gap (simple: "What relates X to Y?")
     b. Execute query (stub for Phase 5: return mock results)
   - Construct GapFillResult with gaps and synthetic queries
   - Return results

6. Private #integrateGapFillResults(results: GapFillResult): void
   - For each synthetic query result:
     a. Extract entities/relations identified
     b. Add new nodes to tnaGraph (entity identifiers)
     c. Add edges for new relations (entity1 — relation_type → entity2)
   - Log: `[ORCH-OBSTRUCTION] Integrated {N} new entities from gap fill`

7. async shutdown(): Promise<void>
   - Drain obstruction queue (process all pending)
   - Shutdown agentPool
   - Unsubscribe from events

8. Helper: getProcessingStatus(): { isProcessing: boolean; queueLength: number; agentCount: number }
   - Return current state (for testing/monitoring)

Key implementation notes:

- Obstruction handling is ASYNCHRONOUS and QUEUED (serialize to maintain ordering)
- Agents are pooled and reused (GapDetectorAgent is a stub for Phase 5)
- Gap fill results are INTEGRATED BACK into TNA graph (feedback loop)
- This implements the obstruction-driven reconfiguration criterion
- No actual LLM calls (Phase 5 is synthetic; Phase 6 would add real catalyst generation)
