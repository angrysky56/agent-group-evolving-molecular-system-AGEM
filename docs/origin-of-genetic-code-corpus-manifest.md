# Corpus Manifest — Origin of the Genetic Code
*An open-discovery run for AGEM. No planted conflict, no answer key.*

## Why this corpus
The biosemiotics run had the right *shape* — rival schools making claims about
shared entities, with a physical substrate available as an external index. Its
only failure was vacuous encoding (every conflict dodged through an empty
domain). This corpus keeps that shape but changes two things: the problem is
genuinely **unsolved** (no scholarly verdict to grade against), and the encoding
is built for **discovery**, not for confirming a contradiction we already know.

In concrete biochemistry this is the origin-of-the-*index* problem: when and why
a physical codon↔amino-acid correlation becomes an arbitrary, conventional
*code*. That framing is for you, the operator — it is **not** to be encoded as a
block. AGEM should find whatever structure is actually there, not inherit a thesis.

## Design rules (what makes this "into the unknown")
1. **No hand-authored opposing blocks.** AGEM extracts blocks from the source
   texts over a shared, neutral vocabulary. We do not pre-encode who disagrees
   with whom.
2. **No answer key.** The problem is open; there is no target verdict. Success is
   judged on epistemic honesty + world-contact (below), never on matching a
   predetermined conclusion.
3. **Existence-commitment discipline — the one thing carried over.** For every
   entity the corpus genuinely commits to, assert `exists x (…)`. This is
   *honesty*, not a crossed signal: applied uniformly to all blocks it favours no
   hypothesis; it only stops "consistent" from meaning "true of an empty world."
   This is the fix from the last run, used correctly.
4. **Indices provided, not weighted.** A set of real touchstones AGEM *may* test
   blocks against. We do not tell it which a block must satisfy, or which index
   decides.

## Shared ontology (neutral — no claims baked in)
A common vocabulary so extracted blocks quantify over the same things:
- `codon(x)`, `amino_acid(x)`, `assignment(c, a)` — codon `c` maps to amino acid `a`
- `stereochemical_affinity(c, a)`, `aptamer_binding(a, triplet)`
- `biosynthetic_precursor(a1, a2)`, `synthetase_class(a, k)` — k ∈ {I, II}
- `error_value(code, v)`, `arbitrary(assignment)`, `conventional(code)`, `frozen(assignment)`
- `ribozyme(x)`, `prebiotic(x)`

Uniform existence seeds (honesty, not a thesis):
`exists x codon(x)`, `exists x amino_acid(x)`, `exists c exists a assignment(c, a)`,
and one for each entity a block actually invokes.

## Source set (balanced across the live hypotheses — pull the real texts)
Feed the primary literature; do **not** pre-summarise it into positions — let
ingestion build the blocks.
- **Frozen accident / contingency** — Crick (1968).
- **Stereochemical / direct affinity** — Yarus et al. (RNA aptamer–amino-acid binding); early Woese.
- **Coevolution (biosynthetic pathways)** — Wong (1975); Di Giulio.
- **Adaptive error-minimisation** — Freeland & Hurst (1998); Haig & Hurst.
- **Open-status syntheses** — Koonin & Novozhilov, "the universal enigma" (2009; 2017 update).

## Index set (touchstones AGEM may adjudicate against — unranked)
- The actual standard codon table and its structure: third-base wobble, synonymy,
  physicochemical clustering of similar amino acids in codon space.
- The **error-minimisation statistic** — the code's rank against random alternative
  codes. A *computable* formal index (the closest analog to the "internalizable
  index" we discussed).
- The aminoacyl-tRNA synthetase **Class I/II** split vs. code structure.
- Aptamer binding-frequency data (stereochemical evidence).
- The amino-acid **biosynthetic precursor–product graph** (coevolution evidence).

## Open questions to pose (genuinely unresolved)
- Is the code's structure better explained by affinity, biosynthesis, selection,
  or contingency — or is it **under-determined** by the available evidence?
- Where do hypotheses make incompatible claims about the **same** entities (real
  conflict), and where are they inter-translatable re-descriptions (notational)?
- Is there a regularity in the code that **none** of the standing hypotheses
  predicts? (the room for genuinely new ground)

## Success criteria (honesty + world-contact, not a key)
A finding is creditable only if it:
1. is grounded in at least one index (it touches the world, not just the texts);
2. survives the consistency layer **with existence asserted** (not a vacuous pass); and
3. states its uncertainty boundary.

"Under-determined" is a legitimate and valuable result here — but only as a *real*
verdict (existence asserted, indices consulted), never as the empty-world artifact.

Log as red flags: any block that passes only vacuously; any conclusion citing no
index; any harmonisation that dissolves a conflict without a formal or empirical
reason. These are the Comfort-Trap signatures at the corpus level.
